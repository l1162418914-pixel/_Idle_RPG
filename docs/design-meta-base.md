```
┌─────────────────────────────────────────┐
│  局外成长（Meta）                        │
│  账号级 / 大营级：解锁、等级、科技、名册  │

│  BASE 态循环：建筑、养伤、背包、编组、后勤 │
└──────────────────┬──────────────────────┘
                   │ start_run

```

### 3.4 角色成长 · 猎杀对决（定方向 · 机制待填）

> **职责**：只管 **局外的人怎么长大、怎么构筑**；不管跑图壳子（§五 在 GAME_BIBLE）与物资箱（塔科夫层）。

#### 猎杀在做什么

| 猎杀机制 | 玩家感受 |
|----------|----------|
| **猎人 = 持久角色** | 不是纯账号数值，是 **有名字、有构筑** 的人 |
| **等级 + 经验** | 局内行为涨 XP；**撤离成功** 才踏实带走成长 |
| **特质 Traits** | 正负特质拼 Build，有槽位上限 |
| **天赋 Perks** | 等级/点数解锁，改玩法而不只 +5 攻击 |
| **出战装备** | 出征前配装；局内丢了很痛 |
| **猎人死亡** | 失败可 **永久失去这名猎人**（比单局死亡更重） |

#### TBH 对照（建议借鉴方式）

| 维度 | 猎杀 | TBH 现网 | 借鉴定案（方向） |
|------|------|----------|------------------|
| 持久单位 | 猎人 | `Mercenary` 名册 + 双半组 | ✅ 以 **佣兵** 为成长载体 |
| 等级/经验 | 局内涨、撤出结算 | `level` / `exp`、结算 `apply_run_rewards` | ✅ **完整探险** 为闸；多源构成（§3.5） |
| 构筑 | Traits + Perks | 技能被动、装备、**伤痕** | ⚠️ 朝 **特质槽 + 被动天赋** 扩（伤痕可类比负特质） |
| 出战配装 | 买装带进图 | PREPARE + 半组 + 箱/外露 | ✅ 已有；强化「出征前整备」权重 |
| 死亡代价 | 猎人没 | 濒死、永久死亡；**目标**鸭科夫式遗留+回收 | ⚠️ 见 [design-failure-lineage.md](design-failure-lineage.md) |
| 账号 Meta | 血点、解锁特质池 | 金币、建筑、地图解锁 | **货币系统从长计议**（§3.6） |

#### 建议借什么 / 不借什么

| 借 | 不借 |
|----|------|
| **佣兵是有身份的 Build**，不是纯数值球 | 1v1 PvP、枪法 |
| **完整探险结算** 为经验生效闸；路上多行为计入经验池 | 每次死亡必删 merc（失败掉人规则另定 §3.6） |
| **正/负特质** 或 **天赋点** 塑造差异 | 猎杀完整特质商店 UI 一键照搬 |
| 出征前 **配装 + 选人** 有仪式感 | 局内翻箱搜枪循环 |

#### 与现网系统的衔接点

| 现网 | 可演进为猎杀味 |
|------|----------------|
| `design-near-death.md` 伤痕 | 负特质 / 长期 debuff，医疗室「洗特质」 |
| `design-expedition-meta.md` 半组 | 固定小队 = 你的「猎人 roster」 |
| 技能 `active_skills` / 被动 | Perk 树或槽位扩展 |
| 永久死亡 `mark_permanent_death` | 仅 **灭团/撤离失败档** 等重罚，非常态 |

#### 拍板记录

| 问题 | 定案 | 状态 |
|------|------|------|
| **经验主闸** | **以完整进行一次探险为基础**；路上多源 + 结算撤离价值共同决定经验量 | ✅ 见 §3.5 |
| 特质系统 | A 新做 trait 槽 B 先扩伤痕/被动 | 待定 |

### 3.5 经验主闸 · 定案（2026-06-05）

#### 原则

1. **闸**：经验在 **完整探险结束并进入结算** 时入账（`RESULT` → `apply_run_rewards` 链路）；以 **「跑完一趟、成功结算档」** 为前提，不是路边即时升级。  
2. **构成**：本趟总经验 = **基础上** + **路上累积池**，结算时按撤离结果一次性分配给人。  
3. **与人绑定**：经验进 **出战佣兵**（及主角规则待定），猎杀式「谁在场谁吃」可在公式阶段再细。

#### 经验构成（定案方向 · 数值未做）

| 来源 | 对应系统 / 未来系统 | 说明 |
|------|---------------------|------|
| **基础上** | 结算档、`retreat_reason`、是否抵营 | 完整探险的「底分」；通关 / 撤离抵营 / 战败档是否同权 **待公式** |
| **路上战斗** | `CombatController`、击杀、接战次数 | 战斗贡献进池；进军与返程接战均计 |
| **搜索** | [design-march-events.md](design-march-events.md) Auto Search | 行军被动搜索命中计入 |
| **事件** | 同上 March Events | 里程碑 / 稀有事件奖励或惩罚 |
| **追击** | [design-boss-chase.md](design-boss-chase.md) | 追击接战、击退、击杀追击 Boss 等 |
| **结算 · 撤离价值** | `carry_value`、箱+外露、智能带货撤 | 结算时按 **本趟携带价值**（或净带出价值）加成经验 |

#### 建议结算流程（实现时对照）

```
RUNNING 期间：
  各子系统只写 run_xp_ledger（本趟经验池分项），不写 merc.exp

RESULT / apply_run_rewards：
  if 满足「完整探险」闸（抵营/通关结算档 — 细则待与失败档对齐）:
      total_xp = base_completion + combat + search + events + chase + extract_value
      按规则分给参战 merc / 主角
      merc.level / exp 更新
  else:
      战败/斩仓/未达闸档的扣减或归零 — 与 §3.6 失败掉人一并定
```

#### 与猎杀对照

| 猎杀 | TBH 定案 |
|------|----------|
| 局内行为涨 XP，撤离带走 | 局内只 **累积池**，结算闸一次性入账 |
| 击杀、发现、目标 | 战斗 + 搜索 + 事件 + 追击 + **带货价值** |

#### 仍待公式（不阻塞文档）

- 各分项权重、上限、重复击杀衰减  
- Boss 通关直结 vs 返程抵营是否同 **基础上**  
- 主角 `base_exp_multiplier` 与佣兵分配比例

### 3.7 失败掉人 · 猎杀 + 鸭科夫合体（方向定案 · 2026-06-05）

> 全文：[design-failure-lineage.md](design-failure-lineage.md)

#### 合体一句话（2026-06-05 用户拍板后）

**猎杀**：有价 Build + 经验按 MIA 人数冻结。  
**鸭科夫**：死亡→MIA→主城 **短 Run / 一键 / 大价值复活**。  
**TBH**：主角永不 MIA；仅 **放弃搜寻** 永久没；拖捞 2 趟地图点消失仍可主城复活。

#### 三层速查

| 层 | 触发 | 结果 |
|----|------|------|
| L1 濒死 | HP/压力清零→1+1+护盾；**抵营** | 过闸养伤 |
| L2 MIA | 灭团全员 / 撤离失败比例 / 死亡二段式 / 压力判定 | 遗物点 + 经验冻结 |
| L3 永久没 | **仅放弃搜寻** | 构筑沉没 |

#### 与 §3.4 猎杀成长的关系

成长越厚，MIA 越痛 → 回收多档：**A/B 互捞轻罚** / **救援队重罚** / 一键 / 大价值。见 failure-lineage §5.7、§6。

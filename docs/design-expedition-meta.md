## 双半组 4+2（H2 仅基地换班）

| 项目 | 规则 |
|------|------|
| **编制** | 半组 A/B 各 **4 出战佣兵 + 2 替补（可空）**；**主角不占槽**（留营独立卡片）；同一人不可同时在 A、B |
| **本趟** | **不**在半路 A↔B 换班 |
| **组内替补** | 撤离 / 双人濒死可顶替补（组内缓冲）；整组不可用再走半组切换（仅 BASE） |

### 双半组语义（2026-06-06 定案）

存档结构不变：

```json
{
  "active_half": "A" | "B",
  "A": { "active": [4], "bench": [2] },
  "B": { "active": [4], "bench": [2] }
}
```

| 概念 | 字段 / API | 含义 | 谁可改 |
|------|------------|------|--------|
| **编组优先** | `active_half` | 玩家设的「编组/补满/备战席点人」默认目标半组 | 仅 **设为优先**、F1/F2 编组 UI |
| **下趟出征** | `resolve_deploy_half()` | 本趟实际使用的半组（只读解析） | **禁止** UI 直接写 |
| **上趟出征** | `last_deploy_half` | 自动再战 / 轮换记忆 | `start_run` 成功后写入 **实际** 出征组 |

**铁律**：`active_half`（编组优先）≠ 下趟一定出征该半组。顶栏须同时展示两者，例如：

`已设编组优先:B | 下趟出征:A（B 无可用出战·自动改派）`

### 出征解析规则

| 场景 | 规则 |
|------|------|
| **手动出征**（玩家点「出发」） | 若 `active_half` 对应半组 **无** ≥1 名 `can_join_squad()` 的出战位佣兵 → **阻止出征**（错误码 **-7**，文案指引编入 / A↔B 互换 / 补满未编入） |
| **自动连续出征 / rebuild_auto_squad** | 允许 `resolve_deploy_half()` 链：编组优先 → 另一半 → 空；若实际组 ≠ 编组优先 → **toast 一次**「本趟改派」 |
| **start_run 成功后** | 只更新 `last_deploy_half = 实际出征组`；**不得** 用实际组覆盖 `active_half` |

### 补员与名册

| 函数 | 行为 |
|------|------|
| **rebalance_from_roster**（招募/读档） | **不**自动占 A/B 槽；新佣兵留在 **备战席 / 未编入**（FORM-3R） |
| **auto_fill_half(half)**（补满优先半组） | 只从 **未编入名册** 且 **不在另一半组** 的佣兵补位；**不**从 A 槽位搬到 B |
| **跨半组搬人** | **A 槽拖入 B 空槽**、点选、或 **A↔B 互换**；禁止静默抢人（FORM-6） |

### 养伤锁与备战席

| 项目 | 规则 |
|------|------|
| **养伤锁** | A、B 两半组 **出战位** 均无可 `can_join_squad()` 佣兵 → 禁出征（主角不参与判定） |
| **养伤锁下** | **可**编组（备战席 → 半组槽位）；**不可**出征 |
| **备战席** | 满血佣兵在备战席 **不能** 单独解除养伤锁；须编入某半组 **出战位** 且满足 `can_join_squad()` |
| **文案** | 备战席「可编入」≠ 出战位「可出征」；出战须 ≥70% HP 等（与 `can_join_squad` 一致） |

## 全队养伤锁

| 项目 | 规则 |
|------|------|
| **触发** | A、B 两半组 **佣兵** 均无可出战成员（**主角状态不参与**） |
| **期间** | 禁再次出征/自动循环；**养伤 UI**；地图 ID 保留 |
| **解除** | **任一半组** 四出战全员 `can_join_squad()`（**70% HP** 清濒死，见 near-death 文档） |
| **恢复** | 基地慢回；养伤中 **医疗室加速**；医护 **优先治疗最快能满队的一组** |

## 出征失败提示

| 返回码 | 含义 |
|--------|------|
| **-1** | 解析后出战名单为空 |
| **-2** | `WorldRun.start` 失败 |
| **-3** | （现网其他校验，见 `get_run_start_error_message`） |
| **-5** | 全队养伤锁 |
| **-7** | **编组优先半组无可用出战**（T-UI-FORM-2）：提示编入出战位 / A↔B 互换 / 补满未编入 |

基地/编队 UI 统一走 `get_run_start_error_message` / 顶栏 `_status_label`。

## 出征策略（大营 F2 · 出发时锁定）

玩家在 **大营编队** 设定本趟目标；`WorldRun.start()` 快照策略，**本趟途中不随大营改动**。逻辑在 `expedition_strategy_service.gd`。

### 推图 `push`

- **目标**：击杀 Boss → 解锁下一关（见 `MapUnlockService` / 结算 `boss_defeated`）
- **撤离**：**仅玩家手动撤离**；搜索到高价值、背包满均**不**自动撤
- **节奏**：略快推进、略少搜索、略多接战（为冲 Boss 服务）

### 搜刮 `loot`

- **目标**：资源优先；更密自动搜索、更少随机接战（**主动避战**待后续 TASK）
- **撤离**：安全箱+外露**装满**或拾取**高价值**装备/物资 → **自动撤离**
- **战利品**：仍可用「挤占低值 / 溢出丢弃」开关（大营勾选）

### 均衡 `march`

- **目标**：正常跑图；推进/搜索/接战均为基准倍率
- **撤离**：携带价值达标或背包将满（`auto_retreat_rules.json`）→ 可自动撤
- **选项**：仅均衡模式显示「达标撤 / 仅安全箱」

### 战利品开关（三模式通用）

| 开关 | 字段 |
|------|------|
| 挤占低值 | `loot_auto_evict_low_value` |
| 溢出丢弃 | `loot_discard_overflow` |

### 节奏倍率（辅助，非策略本体）

| 模式 | 推进 | 搜索间隔 | 遇敌间隔 |
|------|------|----------|----------|
| 推图 | ×1.15 | ×1.25（更少） | ×0.90（更密） |
| 均衡 | ×1.00 | ×1.00 | ×1.00 |
| 搜刮 | ×0.88 | ×0.68（更密） | ×1.28（更少） |

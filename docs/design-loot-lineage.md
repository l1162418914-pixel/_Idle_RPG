# 战利品 · 安全箱 — 设计血统（塔科夫 / 三角洲）

> **用途**：PM / 开发参考。说明 **安全箱 vs 外露** 的借鉴来源与 TBH 定稿。  
> **总纲**：[GAME_BIBLE.md](GAME_BIBLE.md) §五 · 玩法分册：[design-retreat.md](design-retreat.md) §四。

---

## 一、借鉴来源

| 来源 | 借什么 | 不借什么 |
|------|--------|----------|
| **Escape from Tarkov** | 局内 **安全箱/口袋** 与 **身上可丢** 分离；二维占格；撤离才真正「带出」 | 第一人称、部位伤害、硬核操作、战局外市场全复刻 |
| **三角洲行动**（撤离类） | **高价值带货撤**、外露风险、战备与收益权衡语气 | 现代军事 UI 全抄、多人匹配 |

**TBH 合成**：单次 Run = 一局「撤离行动」；**安全箱** ≈ 保险箱格；**外露** ≈ 背包/胸挂易丢区；抵营结算 = 撤离成功进大营仓库。

---

## 二、TBH 定稿对照

| 维度 | 塔科夫/三角洲气质 | TBH 现网 |
|------|-------------------|----------|
| 绝对安全区 | 保险箱、口袋 | `safe_loot` 安全箱格（医疗升级） |
| 可丢区 | 背包、胸挂死后/撤离失败丢 | `exposed_loot` 外露格 |
| 返程风险 | 路上被打掉货 | 双池盾破 → `retreat_hit_drop_chance` 抽外露 |
| 手动放弃 | 丢包跑路 | `manual` 斩仓：仅箱+随身 |
| 智能撤 | 赚够就跑 | `auto_value` / `carry_value` |
| 形状 | 不规则占格 | 1×1 / 1×2 / 2×1 / 2×2（首期） |

---

## 三、开发约束

1. 安全箱内物品 **不可** 被 `retreat_hit`、斩仓（已确认舍弃外露）以外逻辑抽出。  
2. 结算前战利品 **不得** 直接 `InventorySystem.add`（ARCHITECTURE §四）。  
3. UI 右窗 RUNNING 须能一眼区分 **箱 vs 外露**（提灯式风险语气见 GAME_BIBLE）。
